## What is Infobase?

**Infobase** is OneInbox's knowledge management layer. It stores information from **websites** and **uploaded documents** in **folders**, so **AI voice agents** can use that content during live phone conversations.

<Note>
  Infobase is organized into **folders**. Each folder can contain website URLs and uploaded files. You attach an entire folder to one or more agents — not individual files one-by-one on the agent.
</Note>

### What Infobase is

- A **folder-based knowledge library** for your agents
- A way to combine **web pages** and **files** in one place
- A bridge between your content and **agent behavior on calls**

### What Infobase is not

- A document management system (DMS) with in-app editing
- A wiki or CMS where you edit text inside OneInbox
- A replacement for your CRM or ticketing system

## How Infobase works

```mermaid
flowchart LR
  A[Create folder] --> B[Add website or file]
  B --> C[System indexes content]
  C --> D[Status: Ready]
  D --> E[Attach folder to agent]
  E --> F[Agent uses knowledge on calls]
```

1. You **create a folder** in the Infobase screen.
2. You **add sources** — website URLs or uploaded files.
3. The **system crawls or parses** content and builds a searchable index (**In Progress** → **Ready**).
4. You **attach the folder** to an agent in the Agent **Role** tab.
5. During **live calls**, the agent retrieves relevant knowledge from that folder.

You do not manually "train" a model in Infobase. You **supply sources** and wait until they are **Ready**.

## Frontend vs backend

| Activity | Frontend (you in OneInbox) | Backend / system (automatic) |
| --- | --- | --- |
| Create or delete folders | Yes | — |
| Add websites, select pages to index | Yes | Crawls and indexes URLs |
| Upload files (PDF, DOCX, TXT, CSV, PPTX) | Yes | Parses and indexes files |
| Edit uploaded file content in the UI | **No — not on roadmap** | — |
| Update a file after external edit | **Re-upload** the file | Re-indexes (**In Progress** → **Ready**) |
| Check processing status | Yes — view list and **↻ refresh** | Runs processing |
| Refresh website content after site changes | **Manual refresh only** — no auto re-crawl | Re-processes when you refresh |
| Assign users to a folder | Yes (optional) | Access per your org settings |
| Attach folder to an agent | Yes — Agent → Role tab | Uses index on calls |
| Delete a file or folder | Yes | Removes indexed content |

## Key terms

| Term | Meaning |
| --- | --- |
| **Folder** | A container for websites and files (e.g. "Business") |
| **In Progress** | Source is still being crawled or indexed |
| **Ready** | Source is indexed and available to agents |
| **Agent** | Your AI voice configuration in OneInbox |
| **Attachment** | Linking an Infobase folder to an agent on the Role tab |
